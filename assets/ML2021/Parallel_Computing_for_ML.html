Parallel Computing for Machine Learning

Recorded by 2022/6/3

Motivation

  • 目前最流行的机器学习的方法是神经网络，NN存在“大力出奇迹”的趋势，通常规模很大，例如ResNet-50有25M个参数。随着NN加深，模型范式的复杂化，为达成较好的模型效果，参数量暴增。

  • 大模型要在“大数据”上训练，例如ImageNet有14M张图片。在CV、NLP中的数据普遍都这么大

  • 大数据 + 大模型 产生巨大的计算资源开销（e.g. ）

    由于超参数需要调节，单行的训练过程需要反复进行。

  • 因此需要一个并行计算（Parallel Computing）方法：通过多处理器来让计算速度（wall-clock time）加快。

    并行计算只会减少钟表时间，但不会减少GPU、CPU单个计算时间。

Communication

如何进行数据的划分以及参数和梯度结果的传递等等

涉及到处理器之间的通信问题

communication way 1：share memory

处理器之间共享内存（memory），一个处理器可以看到其他处理器的计算结果。局限性：无法做到大规模的并行计算，一个主机的CPU接口是有限的（内核数量有限），无法用数以十记的GPU或CPU做并行计算。

communication way 2：Message Passing

整个并行计算系统有多个节点，每个节点都有多个处理器，节点内可以共享内存，节点之间无法共享内存。节点间可以用网线连起来，也可以远程连接用TCP/IP来收发消息。信息（比如说表示为vector）打包为多个package通过TCP/IP协议进行传输。

如果有多个节点就必须使用message passing，不能使用共享内存。并行计算中，一个核心问题在于如何去协调这些节点工作。以下介绍几种节点间协作的架构（Architecture）
